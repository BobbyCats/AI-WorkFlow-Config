# 调试提示词模板

## 问题诊断

```
这里出了问题，请帮我诊断：
1. 分析可能的原因
2. 按可能性排序
3. 给出验证每个假设的方法
4. 不要直接修复，先确认根因
```

## 自动修复

```
去修复这个问题。
- 不需要我告诉你怎么做
- 修复后告诉我你做了什么
- 如果需要更多信息，自己去查
```

## 对比验证

```
向我证明这个修复是有效的：
1. 解释改动的原理
2. 对比修复前后的行为
3. 说明为什么这样修复是正确的
```

## 迭代修复

```
实现后，运行测试并告诉我结果。
如果失败，自行修复。
重复这个过程，直到所有测试通过。
每次迭代告诉我你做了什么改动。
```

## 日志分析

```
分析这些日志/错误信息：
1. 提取关键错误
2. 追踪错误来源
3. 识别模式或趋势
4. 建议排查方向
```
